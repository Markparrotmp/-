# 🏐 Волейбольный бот для Телеграма

Бот присылает в беседу опрос **каждый день в 12:00 по Москве** с тремя вариантами:

- Да ✅
- Нет ❌
- 50 на 50 🤔

Голосование не анонимное — видно, кто и как проголосовал.

---

## 🚀 Рекомендуемый способ: бесплатно, без своего сервера (GitHub Actions)

Опрос будет отправлять сам GitHub раз в день по расписанию. Ничего держать
включённым не нужно, терминал не требуется — только браузер. Всё настраивается
за 5 минут.

### Шаг 1. Создать бота
1. Открой в Телеграме [@BotFather](https://t.me/BotFather), отправь `/newbot`,
   придумай имя и username.
2. Сохрани **токен** вида `123456789:AAE...`.

### Шаг 2. Добавить бота в беседу и узнать её ID
1. Добавь бота в волейбольную беседу.
2. Напиши в беседе любое сообщение с командой, например `/start`.
3. Открой в браузере ссылку (подставь свой токен вместо `ТОКЕН`):
   `https://api.telegram.org/botТОКЕН/getUpdates`
4. В открывшемся тексте найди `"chat":{"id":-100...}` — это число (обычно с
   минусом) и есть **ID беседы**. Скопируй его.
   > Эту ссылку никому не показывай — в ней твой токен.

### Шаг 3. Прописать токен и ID в секретах репозитория
1. На странице этого репозитория на GitHub: **Settings → Secrets and variables
   → Actions → New repository secret**.
2. Создай два секрета:
   - `BOT_TOKEN` — токен из шага 1
   - `CHAT_ID` — число из шага 2

### Шаг 4. Включить и проверить
1. Открой вкладку **Actions**. Если Actions выключены — нажми кнопку включения.
2. Слева выбери воркфлоу **«Волейбольный опрос»** → справа **Run workflow** →
   зелёная кнопка. Через несколько секунд опрос должен прийти в беседу.
3. Готово! Дальше опрос будет приходить автоматически каждый день в 12:00 МСК.

> ⚠️ **Важно:** автоматический запуск по расписанию работает только когда файл
> `.github/workflows/volleyball-poll.yml` находится в основной ветке `main`.
> Если ты работаешь в отдельной ветке — слей её в `main` (кнопка ручного запуска
> работает из любой ветки, а вот расписание — только из `main`).

> ℹ️ Два нюанса GitHub Actions: запуск по расписанию может опоздать на несколько
> минут (для волейбола не критично); и если в репозитории 60 дней не было
> никаких изменений, GitHub приостанавливает расписание — любой коммит снова его
> включает.

### Как поменять время
В файле `.github/workflows/volleyball-poll.yml` строка `cron: "0 9 * * *"` —
это 12:00 МСК (09:00 UTC). Формат: `минута час * * *`, время в **UTC** (Москва =
UTC+3). Например, 18:00 МСК = 15:00 UTC → `0 15 * * *`.
Текст и варианты опроса — в начале файла `send_poll.py`.

## Альтернатива: запустить как обычную программу у себя (нужен постоянно включённый компьютер/сервер)

Этот вариант умеет ещё и команды `/chatid` и `/poll`, но работает, только пока
запущен процесс.

## Что понадобится

- Компьютер или сервер, где Python 3.9+ и есть возможность держать процесс запущенным (бот работает, только пока запущен).
- Аккаунт в Телеграме.

## Шаг 1. Создать бота и получить токен

1. Открой в Телеграме [@BotFather](https://t.me/BotFather).
2. Отправь `/newbot`, придумай имя и username.
3. BotFather пришлёт **токен** вида `123456789:AAE...`. Сохрани его.

## Шаг 2. Добавить бота в беседу

1. Добавь своего бота в волейбольную беседу.
2. Дай ему право отправлять сообщения (в группах по умолчанию есть).
3. Чтобы бот видел команды в группе, у @BotFather можно отключить Group Privacy
   (`/setprivacy` → выбрать бота → `Disable`). Для отправки опросов это не обязательно,
   но команды `/chatid` и `/poll` в группе будут работать надёжнее.

## Шаг 3. Установить зависимости

```bash
pip install -r requirements.txt
```

## Шаг 4. Узнать ID беседы

Токен уже есть, ID чата — ещё нет. Узнаём его так:

```bash
export BOT_TOKEN='ВАШ_ТОКЕН'   # Windows PowerShell: $env:BOT_TOKEN='ВАШ_ТОКЕН'
python bot.py
```

Бот запустится. Теперь в беседе отправь команду `/chatid` — бот ответит числом
(для групп обычно с минусом, например `-1001234567890`). Скопируй его и останови
бота (`Ctrl+C`).

## Шаг 5. Запустить бота по-настоящему

Задай обе переменные и запусти:

```bash
export BOT_TOKEN='ВАШ_ТОКЕН'
export CHAT_ID='-1001234567890'
python bot.py
```

Готово. Пока процесс работает, каждый день в 12:00 по Москве в беседу будет
приходить опрос.

> Можно скопировать `.env.example` в `.env`, вписать туда значения и загружать их
> удобным способом (например, через `python-dotenv` или `export $(cat .env | xargs)`).

## Команды бота

| Команда   | Что делает                                                  |
|-----------|-------------------------------------------------------------|
| `/start`  | Приветствие и справка                                       |
| `/chatid` | Показывает ID текущего чата (нужно для настройки)            |
| `/poll`   | Отправляет ежедневный опрос прямо сейчас                     |
| `/ab`     | Отправляет опрос «аб» (Да / Нет / 50 на 50)                  |
| `/pest`   | Отправляет опрос «пест» (Да / Нет / 50 на 50)                |
| `/pursh`  | Отправляет опрос «пурш» — без прогноза погоды                |

Новые опросы добавляются одной строкой в словарь `EXTRA_POLLS` в `bot.py`
(вторым значением указывается, добавлять ли к опросу прогноз погоды).

## Погода в опросах

К каждому опросу автоматически добавляется прогноз (Open-Meteo, бесплатно, без
ключей): температура и осадки к началу игры, например
«Погода к 17:30: +23° ☀️ ясно». Если сервис погоды недоступен, опрос приходит
без прогноза. Город и час игры настраиваются константами `WEATHER_*` в `bot.py`.

## Как поменять время или текст

Всё в начале файла `bot.py`:

```python
QUESTION = "Играем сегодня в волейбол? 🏐"
OPTIONS = ["Да ✅", "Нет ❌", "50 на 50 🤔"]
TIMEZONE = ZoneInfo("Europe/Moscow")
POLL_TIME = time(hour=12, minute=0, tzinfo=TIMEZONE)
```

Поменяй `hour`/`minute` — и перезапусти бота.

## Запуск на VPS (круглосуточно, с командами)

Нужен любой дешёвый VPS с Ubuntu/Debian (самый маленький тариф подойдёт — боту
хватит 512 МБ памяти). После аренды у тебя будут IP-адрес, логин `root` и пароль.

Подключись к серверу по SSH (в Windows — открой PowerShell и введи
`ssh root@IP_СЕРВЕРА`, потом пароль) и выполни по очереди:

```bash
# 1. Установить Python и git
apt update && apt install -y python3-venv git

# 2. Скачать проект
# Репозиторий приватный, поэтому git спросит логин и пароль:
# логин — имя аккаунта GitHub, пароль — персональный токен (не пароль от сайта!).
# Токен создаётся здесь: https://github.com/settings/tokens (Generate new token
# (classic), галочка "repo"). Либо сделай репозиторий публичным — тогда просто:
git clone https://github.com/Markparrotmp/-.git /opt/volleybot
cd /opt/volleybot

# 3. Установить зависимости в отдельное окружение
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 4. Создать файл с токеном и ID чата
cp .env.example .env
nano .env        # впиши свои BOT_TOKEN и CHAT_ID, потом Ctrl+O, Enter, Ctrl+X

# 5. Установить как службу — бот будет запускаться сам, даже после перезагрузки
cp volleybot.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now volleybot

# 6. Проверить, что работает
systemctl status volleybot     # должно быть зелёное "active (running)"
journalctl -u volleybot -f     # живые логи бота (выход — Ctrl+C)
```

Готово: бот на сервере отвечает на команды и сам шлёт ежедневный опрос в 12:00 МСК.

> ⚠️ **Когда бот на VPS заработал** — отключи расписание в GitHub Actions, иначе
> ежедневный опрос будет приходить дважды (от сервера и от GitHub). Для этого
> удали блок `schedule:` из `.github/workflows/volleyball-poll.yml` или отключи
> воркфлоу на вкладке Actions (⋯ → Disable workflow).

Обновление бота на сервере после изменений в репозитории:

```bash
cd /opt/volleybot && git pull && systemctl restart volleybot
```
